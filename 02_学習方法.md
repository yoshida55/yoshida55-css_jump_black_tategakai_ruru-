# コーディング学習 - 一番力がつく流れ

---

## ステップ 1：仕様を把握する（10 分）

- 仕様書を読む
- Figma でざっと全体を見る
- 「何が必要か」を頭に入れる

---

## ステップ 2：HTML 構造を先に全部書く（30 分）

- CSS は一切書かない
- セマンティックなタグを意識（`header`, `main`, `section`, `footer`）
- クラス名もこの時点で決める

**→ ここで「設計力」がつく**

---

## ステップ 3：CSS をセクションごとに書く（メイン作業）

- 上から順番に 1 セクションずつ
- PC 版を先に完成させる
- 1 セクション終わったらデモと見比べる

**→ ここで「実装力」がつく**

---

## ステップ 4：レスポンシブ対応

- 900px 以下の調整
- PC 完成後にまとめてやる

---

## ステップ 5：答え合わせ（重要）

- 自分のコードとソースコードを比較
- 「なぜ違うのか」を考える
- 良い書き方があればメモする

**→ ここで「改善力」がつく**

---

## やりがちな NG

| NG                                   | なぜダメか               |
| ------------------------------------ | ------------------------ |
| 最初からソースコード見る             | 考える力がつかない       |
| HTML 書きながら CSS 書く             | 構造がぐちゃぐちゃになる |
| デモサイト見ながら DevTools でコピー | 何も身につかない         |
| 詰まったらすぐ答え見る               | 調べる力がつかない       |

---

## 詰まったときの順番

1. Figma をもう一度よく見る
2. 「CSS ○○ やり方」でググる
3. それでも無理なら**レイアウト構成ページ**を見る（ヒント）
4. 最終手段でソースコード

**この順番を守ると、自走力がつく。**
